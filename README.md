# ci-cd-runbooks
This repository serves as the central knowledge base for designing, implementing, and managing complete CI/CD pipelines. It contains step-by-step runbooks, best practices, and reference architectures covering the entire software delivery lifecycle — from code commit to production deployment.

### What's Inside

- **CI/CD Fundamentals** – Concepts, benefits, and core architectural patterns
- **Jenkins** – Master/agent setup, pipeline as code, plugin management, and security hardening
- **SonarQube** – Code quality gate configuration, static analysis integration, and reporting
- **ArgoCD** – GitOps principles, application deployment, sync policies, and cluster management
- **Kubernetes** – Cluster setup (on-prem/cloud), namespace strategies, RBAC, and ingress controllers
- **Build Servers** – Configuration, resource optimization, and toolchain integration
- **End-to-End Pipeline Examples** – Real-world workflows connecting all tools together

### Purpose

- Standardize CI/CD implementation across teams
- Reduce onboarding time for new platform engineers
- Provide battle-tested runbooks for troubleshooting
- Serve as a single source of truth for DevOps practices
